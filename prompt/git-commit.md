📝 **提交规范（Commit Style Guide）**

🌐 **语言**：必须使用**中文**

🏗️ **结构**：遵循 **Conventional Commits** 规范

```
<类型>(<作用域>): <主题>

- <详细变更描述 1>
- <详细变更描述 2>
...
```

### 📌 头部（Header）说明：

- **类型**：使用以下类型之一：
  - `feat`：新功能
  - `fix`：修复问题
  - `refactor`：重构
  - `docs`：文档更新
  - `style`：代码格式调整
  - `test`：测试相关
  - `chore`：构建/依赖管理等杂项

- **作用域**：（可选）变更影响的包或模块名，如 `clog`、`connector`、`lock`。如果影响全局或不确定，可省略

- **主题**：对变更内容的简短描述
  - 使用**祈使语气**（例如："添加功能" 而不是 "添加了功能"）
  - 首字母小写
  - 不要使用句号结尾

### 📄 正文（Body）说明：

- 如果变更包含多个逻辑点，**必须**提供正文部分
- 使用 `-` 作为列表项开头
- 描述**"做了什么"**以及**"为什么做"**，保持简洁清晰
